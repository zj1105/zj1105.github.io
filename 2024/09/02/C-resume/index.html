C++ 简介

C++特点

1、继承自C语言 : X++是在C语言的基础上发展过来
的，继承了C语言简洁、紧凑、使用方便、灵活的特点。
2、面向对象和面向过程
3、编译器严格，与C语言相比、C++的编译器更加的严格
4、扩展性：

面向对象的特点

封装，继承，多态

构造函数

定义，一种特殊的成员函数，在类被创建时自动调用，目的主要是初始化状态。

析构函数

C++中的一种特殊的成员函数，当对象的声明周期结束时，析构函数会被自动调用。主要的作用是用来资源清理，释放内存，关闭打开的文件等等。

拷贝构造函数

是一个特殊的构造函数，用于创建一个特殊的新的对象作为现有对象的副本，当一个新的对象被初始化为已存为对象时，拷贝构造函数会被自动调用。

深拷贝：复制执政的值还会复制指针指向的对象。
浅拷贝：默认浅拷贝，只会复制指针，两个对象指向同一块内存。

拷贝构造函数作为参数传递
从函数返回对象。

引用

引用必须在声明的时候初始化，初始化过后不能再改变它所引用的对象，引用没有自己地址，无法通过从新复制来改变引用所指向的对象。
默认拷贝构造函数的行为：对于应用类型的成员变量，默认拷贝构造函数不会创建新的引用，也不会改变原有的引用。

静态成员数据

类的静态成员数据时类的一个属性，不属于类的任何实例，与类本身关联，无论创建多少个类的对象，静态成员数据都只有一个共享的拷贝，生命周期贯穿整个程序的执行过程。

静态成员函数

特殊的成员函数，只能访问静态成员变量，或者其他的静态成员函数，不能访问其他非静态成员函数。
因为这些非静态成员是与具体对象实例相关联的。

单例模式

一种设计模式，保证一个类只有一个实例，提供一个全局的访问点来访问这个实例，这样做的目的是节省胸痛资源

懒汉式

饿汉式

变量的生命周期

全局变量：再程序任何都能够访问到，再程序执行期间是有效的
局部变量：定义在函数体内的变量，只能在函数体和代码块内访问，生命周期在函数执行完成。
静态局部变量在函数内定义静态变量，生命周期在整个程序范围内，作用域在函数体内
静态全局变量，作用域在整个文件，生命周期在程序执行范围内。

C++命名空间

避免命令冲突
组织代码：把相关的函数，类封装在一起，提高代码的可读性
模块化开发，可以将代码划分为不同的模块，是的代码更加的易于维护。

include “” 和 <> 的区别

#include<>编译器会按照标准库路径进行搜索  
#include""现在当前目录下进行搜索。

智能指针

智能指针，特殊的指针，用于管理动态分配的内存，能够自动管理指向内存资源的生命周期，防止内存泄漏。
unique_ptr
shared_ptr
weak_ptr

栈和堆的区别

1、分配方式
堆：程序员自动方式
栈：系统自动分配，先进后出，
为什么，访问函数的时候，函数入口地址-》变量。
2、存储方式
堆：从低地址到高地址
栈：从高地址到低地址

栈和队列

栈：先进后出
队列：先进先出

进程

进程和线程的区别

进程是系统中资源分配的基本单位，有自己独立的内存空间和资源等等，线程是进程中的执行单元，同一进程中的线程共享进程的资源，线程的创建和销毁比进程小，但进程有着更好的隔离性和稳定性。

Makefile

TARGET=main
SRC=${wildcard *.c}
OBJ=${patsubst %.c, %.o, ${SRC}}

%.o: %.c

.PHONY: clean
clean: 
    rm *.o ${TARGET}

三次握手

客户端发送一个SYN=1给服务端，服务端收到客户端的请求回复一个SYN=1，ACK=1，客户端收到服务端的ACK，给服务端返回一个确认。

四次挥手

客户端数据发送完毕，发送一个结束连接的信号给服务端，服务端收到信号过后给客户端返回一个ACK，这是客户端已经结束连接，但仍然能够接受数据，服务端数据传输结束过后，释放一个请求结束连接的信号，客户端收到信号过后，回复一个确认，四次挥手完毕。
